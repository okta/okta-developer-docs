* You can add the Entitlement Management capability only if your integration also supports provisioning with API integration actions.

* The API service capability is mutually exclusive of other capabilities in a submission. If you selected the SSO or provisioning capabilities, you can't select the API service capability (the option is unavailable in the submission).

* You can't update the authentication type for a published integration.

* You can't change or remove any existing capabilities or protocols.

* You can't remove existing actions or flows for the operations in your existing public integration. You can only replace actions or flows to your existing operations. You can also add new actions or flows to new optional operations in your integration.
